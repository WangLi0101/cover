| 选择器类型                       | 优先级值 | 描述                             |
| :------------------------------- | :------- | :------------------------------- |
| 内联样式 (Inline styles)         | 1-0-0-0  | 在元素标签中指定的 style 属性    |
| ID 选择器 (ID Selectors)         | 1-0-0    | 使用 id 属性指定的选择器         |
| 类选择器、伪类选择器和属性选择器 | 0-1-0    | 类选择器、伪类选择器和属性选择器 |
| 元素和伪元素选择器               | 0-0-1    | 元素名称本身或元素的伪元素       |
| 通配符选择器 (\*)                | 0-0-0    | 匹配所有元素                     |
| 继承样式 (Inherited styles)      | N/A      | 从父级元素继承的样式             |

### 举例

```html
<div id="test">
  <span>Text</span>
</div>
```

```css
div#test span {
  color: green;
}
div span {
  color: blue;
}
span {
  color: red;
}
```

> 无论 css 语句的顺序是什么样的，文本都会是绿色的（green），因为这一条规则是最有针对性、优先级最高的。（同理，无论语句顺序怎样，蓝色 blue 的规则都会覆盖红色 red 的规则）

> 使用更具体的规则。在你选择的元素之前，增加一个或多个其他元素，使选择器变得更加具体，并获得更高的优先级。

### 计算

```css
[type="password"],
input:focus,
:root #myApp input:required {
  color: blue;
}
```

- `[type="password"]` 是属性选择器，优先级为 0-1-0
- `input:focus` 是元素选择器 + 伪类选择器，优先级为 0-0-1 + 0-1-0 = 0-1-1
- `:root #myApp input:required` 是 伪类选择器 + ID 选择器 + 元素选择器 + 伪类选择器，优先级为 0-1-0 + 1-0-0 + 0-0-1 + 0-1-0 = 1-2-1

### 例外规则

#### `:is()` 、 `:not()` 和 `:has()`

对于 `:is()`、`:has()` 和 `:not()` 伪类，它们本身不会增加选择器的优先级权重(0-0-0)。它们的优先级权重计算不会计入整体优先级值中。相反，这些伪类括号里的参数会根据其自身的优先级值来影响整体选择器的优先级。因此，在优先级值计算时，只有括号内的选择器参数才会起作用，并影响最终的优先级计算结果。

#### `:where()`

优先级调整伪类 `:where()` 总是会将其优先级替换为零（0-0-0）。它使 CSS 选择器能够非常具体地确定目标元素，而不会提升优先级。
