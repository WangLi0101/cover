JavaScript 代码压缩（minify）的主要目的是通过减少代码文件的大小来提高网页加载速度和性能。其原理主要包括以下几个方面：

### 1. **删除多余字符**

- **空白字符**：删除所有不影响代码执行的空白字符，如空格、换行符、制表符等。
- **注释**：删除所有类型的注释，包括单行注释（`//`）和多行注释（`/* */`）。
- **换行符**：删除换行符，将代码压缩成一行。

### 2. **重命名变量和函数**

- **缩短标识符**：将变量名、函数名、参数名等重命名为更短的名称（如将 `longVariableName` 重命名为 `a`），以减少代码长度。
- **作用域优化**：通过作用域分析，仅重命名局部变量和函数，确保全局变量和函数名不冲突。

### 3. **简化表达式**

- **常量折叠**：将常量表达式计算出结果，并替换为该结果（例如，将 `2 + 3` 替换为 `5`）。
- **布尔表达式优化**：简化布尔表达式，如将 `true && x` 简化为 `x`，将 `false || x` 简化为 `x`。
- **条件表达式优化**：将三元表达式和 if-else 语句在可能的情况下进行简化。

### 4. **删除未使用的代码**

- **死代码消除**：删除从未执行的代码片段（如永远不会满足的条件语句）。
- **未引用的代码**：删除未被引用的变量、函数、类等。

### 5. **短路求值**

- **逻辑运算优化**：利用短路求值特性优化代码，例如 `x && y || z`，在可能的情况下删除不必要的部分。

  > `x && y`：如果 `x` 为 `false`，那么表达式的结果就是 `false`，因此 `y` 不会被计算。
  >
  > `x || y`：如果 `x` 为 `true`，那么表达式的结果就是 `true`，因此 `y` 不会被计算。

### 6. **内联代码**

- **内联函数**：将短小的函数直接内联到调用处，减少函数调用的开销。
- **内联常量**：将常量值直接替换为代码中的具体值。

### 7. **其他优化**

- **属性访问优化**：将多次属性访问的对象缓存到局部变量中，减少对象属性访问的开销。
- **字符压缩**：在可能的情况下，使用单字符表示法，如将 `"\n"` 替换为 `"\x0a"` 等。
