### **`display: none;`**

- 元素将完全从文档流中移除，不占据任何空间，页面布局会重新计算。

- 用法：

  ```css
  .hidden {
    display: none;
  }
  ```

### **`visibility: hidden;`**

- 元素依然保留在文档流中，仍然占据空间，但不会被渲染出来，类似“透明”效果, 但作用于其上的事件(如点击)无效。

- 用法：

  ```css
  .hidden {
    visibility: hidden;
  }
  ```

### **`opacity: 0;`**

- 元素的透明度设置为 0，使其不可见，但它仍然保留在页面中，并占据空间，可以进行交互（如点击）。

- 用法：

  ```css
  .hidden {
    opacity: 0;
  }
  ```

### **`position: absolute;` + **`left: -9999px;`\*\*

- 将元素定位到屏幕之外，通常用于视觉上隐藏但仍保留在文档流中的元素（例如用于无障碍设计）。

- 用法：

  ```css
  .hidden {
    position: absolute;
    left: -9999px;
  }
  ```

### **`clip-path: inset(100%);`**

- 使用 `clip-path` 属性将元素裁剪到完全不可见。

- 用法：

  ```css
  .hidden {
    clip-path: inset(100%);
  }
  ```

### **`transform: scale(0);`**

- 将元素缩放到不可见大小，仍然在文档流中保留空间。

- 用法：

  ```css
  .hidden {
    transform: scale(0);
  }
  ```

### **`height: 0;` 和 `overflow: hidden;`**

- 将元素的高度设置为 0，并隐藏溢出的内容，适合用于需要隐藏可展开内容的场景。

- 用法：

  ```css
  .hidden {
    height: 0;
    overflow: hidden;
  }
  ```

### **`font-size: 0;`**

- 将元素的字体大小设置为 0，使其不可见，但仍然保留在页面中。

- 用法：

  ```css
  .hidden {
    font-size: 0;
  }
  ```

### **`content-visibility: hidden;`**

- 使用 `content-visibility` 属性可以隐藏元素的内容，但保留其框架，以提高性能。

- 用法：

  ```css
  .hidden {
    content-visibility: hidden;
  }
  ```

> 最后两个只针对文本内容，不适用于所有元素。
