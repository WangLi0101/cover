##  `<link>` 与 `@import` 的详细比较

| **特性**         | **`<link>` 标签**                                      | **`@import` 规则**                                     |
| ---------------- | ------------------------------------------------------ | ------------------------------------------------------ |
| **使用位置**     | HTML 文档的 `<head>` 部分                              | CSS 文件的顶部                                         |
| **加载方式**     | 并行加载多个 CSS 文件                                  | 顺序加载，后一个文件等待前一个加载完成                 |
| **性能**         | 更高效，减少页面渲染时间                               | 较低效，可能导致页面加载延迟                           |
| **灵活性**       | 支持在 HTML 中动态添加或删除 `<link>` 标签             | 只能在 CSS 文件中静态引入，无法在 HTML 中控制          |
| **条件加载**     | 可以结合 JavaScript 动态添加 `<link>` 标签实现条件加载 | 依赖于 CSS 的 `@media` 规则，灵活性有限                |
| **浏览器兼容性** | 广泛支持，所有现代浏览器均支持                         | 广泛支持，但在旧版浏览器（如 IE6）中存在兼容性问题     |
| **优先级与覆盖** | 依照引入顺序，后引入的样式覆盖前者                     | 同样依照引入顺序，但受限于顺序的单向性                 |
| **维护与组织**   | 适合管理多个独立的 CSS 文件，便于维护和组织            | 适合模块化 CSS，减少文件数量，但可能导致依赖关系复杂化 |
